Technical Changes

Considering that I have more experience with web development than I did last year (the year when this website was first created), I decided to tackle one of the most frustrating problems I experienced with the website: repetition of code. Last year, since I was coding everything solely with plain HTML and CSS, every time I created a new page, I had to copy and paste multiple sections, including—but definitely not limited to—the header and footer for each page. This was not only incredibly tedious to accomplish, but it was also error-prone. Additionally, if I wanted to make a change to the header for instance, I had to go to every single individual page and make that change—no matter how minor—just to keep things consistent across the website. Updating the blog was nightmarish, to say the least. When I added a new blog post, I had to go into every single individual blog post (at the time, there were over 70 separate pages), and change the "recent posts" section. When the page limit of 5 posts was reached, I then had to change the pagination for all of the current pages again. I didn't even think about creating an archive, as these problems would only magnify if I did.

Thus, going into my second year with this website, I knew that I would have to continue maintaining the site. But after examining the current system in place, I decided to turn to Jekyll. Jekyll is a "simple, blog-aware static site generator," but in other words, it just allows me to make my own HTML templates. Now, for example, instead of copying and pasting my header and footer in every single page, I can just put my HTML for these parts into a separate file and include it in my own default template coded using Liquid. Every other page I create then extends this default template. So now, if I make a change to the header, I only need to change one file, and everything else will be updated accordingly. For the blog we have, this is a crucial tool. In fact, I've essentially automated almost everything about the blog, from the pagination to the recent posts column to the archive. Now, if I want to make a new blog post, I only need to put the information in a simple Markdown file, making it much easier to maintain and update the blog. We've also been writing more blog posts just because it's so easy to do!

Here is what I used to make the website this year:

  • HTML (HyperText Markup Language)
  • CSS (Cascading Style Sheets)
  • Javascript
  • Jekyll
  • Liquid
  • YAML

Stock Photos

For some of the pages, we used photos from Pexels, a website which provides free stock photos. Since the photographers that upload their images to Pexels know their images will be reused by others, there are no copyright conflicts by using them.

Starting from Scratch

Hi, I'm Esther, one of the members from team Exothermic Creation (10C), and I created the Exothermic Robotics website! My team members, Ann, Aaryan, Bohan, Helena, and Thomas, as well as the rest of the club helped by gathering information and pictures. My team decided to enter into the Robotics Team Website Challenge sponsored by Dell not only because we wanted to compete in the Online Challenge, but also because Exothermic Robotics has never had a club website before, so we thought that our website could potentially become the official club website.

Research

As mentioned before, since we did not have a previous website, there was very little to go off of. To collect information, my teammates and I interviewed the board members about our club's history and we dug deep into social media and news posts. Unfortunately, the information would often times be scattered and unorganized or nowhere to be found at all. Fortunately though, everyone was very supportive and responded to our questions as soon as possible. My team members would also direct me towards any resources they thought would be helpful!

Design

I used a variety of colors, although the main theme that my teammates and I decided on was a light green, gray/black, and red combination since these are the colors of our T-shirts and logo.

Actual Website Stuff

I only had minimal experience regarding building websites, so building this website certainly seemed daunting at first. However, I decided to take on the challenge and instead of using premade templates from Bootstrap or utilizing website builders such as Wix or Squarespace, I chose to make the website completely from scratch. By watching online tutorials and utilizing other online resources, I was able to gain a better grip of the languages. This is what was used to make the website:

  • HTML (HyperText Markup Language)
  • CSS (Cascading Style Sheets)
  • Javascript

What's Special About Our Website?

Our website is built simply with HTML, CSS, and Javascript! We didn't use any pre-made templates or website builders. While it may have been much harder and more time-consuming to make, it was worth it to learn a new skill that may be useful for the future.

Everyone contributed! All the EXO members provided photos and information or resources regarding the club.

Our website showcases where we are right now, where we were in the past, and where we want to be in the future. As the season goes on, we'll make changes to the website to accurately reflect the club, and hopefully persuade other high school students around the area to join!

Plans for the Website's Future

  • Add a sign-up/login: This will be tricky, but hopefully, by next year, we can get a working sign-up/login system!
  • Continue to maintain the website (fixing bugs, adding new information, etc.): As we attend more events and accumulate awards, the website will be updated accordingly!
